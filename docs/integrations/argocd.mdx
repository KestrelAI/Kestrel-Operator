description: "Connect ArgoCD for GitOps sync actions, deployment tracking, and status monitoring in workflows"

The ArgoCD integration connects Kestrel to your in-cluster ArgoCD installation, enabling AI agents to sync applications, monitor deployment status, and track GitOps operations as part of automated workflows and incident investigation.

## Prerequisites

- Kestrel Operator deployed in your cluster (see [Kubernetes](/integrations/kubernetes))
- ArgoCD installed in the same cluster

## Setup

ArgoCD is auto-detected by the Kestrel Operator when it is installed in the same cluster. No manual connection is required.

<Steps>
  <Step title="Enable ArgoCD in cluster configuration">
    During cluster onboarding or in the cluster settings page:
    1. Navigate to **Integrations → Kubernetes**
    2. Select your cluster
    3. Enable **ArgoCD Integration** in the cluster configuration

    Alternatively, configure via Helm values:
    ```yaml
    operator:
      argocd:
        enabled: true
        namespace: "argocd"
    ```
  </Step>
  <Step title="Verify detection">
    Check the operator logs for ArgoCD discovery:
    ```bash
    kubectl logs -n kestrel-ai deploy/kestrel-operator | grep -i "ArgoCD"
    ```
  </Step>
</Steps>

<Note>
  The Kestrel Operator discovers ArgoCD by looking for the ArgoCD Application CRDs and API server in the cluster. If ArgoCD is installed in a non-default namespace, specify it in the Helm values.
</Note>

## How It's Used

### In Workflows

- **Action blocks**:
  - **Sync Application** — trigger an ArgoCD sync for a specific application
  - **Wait for Sync** — block workflow execution until a sync operation completes (with configurable timeout)
  - **Get Application Status** — retrieve the current sync and health status of an ArgoCD application
  - **List Applications** — list all ArgoCD applications with status summaries

Example: A workflow triggered by a merged PR can automatically sync the corresponding ArgoCD application, wait for the sync to complete, verify health status, and notify the team on Slack.

### In Incident Response

- **Deployment correlation** — during root cause analysis, Kestrel checks recent ArgoCD syncs to determine if a deployment caused the incident
- **Sync status** — surfaces out-of-sync or degraded ArgoCD applications alongside Kubernetes incident data
- **Rollback context** — provides ArgoCD sync history to help identify which revision introduced the issue

### In Cloud AI Copilot

- Query ArgoCD status: `What is the sync status of the payments application?`
- Trigger operations: `Sync the production-api ArgoCD application`
- Investigate deployments: `What ArgoCD syncs happened in the last hour?`

## Disconnecting

To disable the ArgoCD integration:

1. Navigate to **Integrations → Kubernetes** and select your cluster
2. Disable **ArgoCD Integration** in the cluster configuration

Or set `operator.argocd.enabled: false` in your Helm values and upgrade the operator.
